# F1 section — v0 design doc

Mirrors `docs/NFL_V0.md`. Goal: thin first surface for F1 data,
with architecture that grows into Top-5-level complexity without
rewriting.

---

## v0 scope (what we built)

- **12 channels in the DB**: the 11 constructor channels + the
  official F1 HQ channel (@Formula1). All flat — no
  championship / season grouping in the UI.
- **Channel-level only**: subs, total views, total videos,
  last-fetched. **No video ingestion, no per-video snapshots, no
  per-video deltas.** (Same restraint as NFL v0.)
- **One hidden page** at URL `/f1`, not in the sidebar nav.
  Reachable only by direct link.
- A daily channel-stats cron, isolated from Top-5 / WC2026 /
  Players / Women / OtherClub / NFL.

Total new files: 5 (`data/f1_teams.json`, `scripts/import_f1.py`,
`scripts/daily_f1.py`, `src/f1_filter.py`, `views/21_F1.py`).
Existing files touched: 8 (`app.py` + 7 spots in
`src/`/`scripts/` to add `"F1"` to entity-type exclusion tuples).

## Channel taxonomy

```json
"competitions": {
  "f1": {
    "role": "hq" | "team",
    "team": "Oracle Red Bull Racing" | "FORMULA 1",
    "championship": "F1"
  }
}
```

`role == "hq"` is the canonical HQ test (used by `/f1` filter's
"Teams only" mode and the page's HQ chip). `championship` is
forward-compatible if F2 / F3 / F1 Academy channels are added later.

### Shared brand channels

Two teams operate a single brand channel for road cars + F1 instead
of a dedicated F1 channel:

- **Scuderia Ferrari** → @ferrari (2.29M subs, brand-wide)
- **BWT Alpine F1 Team** → @alpinecars (109K subs, brand-wide)

The seed JSON records this in a `notes` field. Stats are inflated vs
pure-F1 channels — kept anyway because they're the team's only
official YouTube presence.

## Forward-compatibility hooks (same as NFL v0)

- **Isolation key** — `entity_type = "F1"` on all 12 channels.
  Added to every entity-type exclusion tuple in src/dashboard_cache,
  src/filters, scripts/{daily,weekly,hourly}_refresh,
  scripts/hourly_rss, scripts/collect_digital_footprint,
  scripts/seed_channel_websites. ~17 line-additions, mechanical.
- **Cohort metadata** — `competitions.f1` stored on every row.
- **Filter widget** — `src/f1_filter.py` with `[All, Teams only,
  HQ, <team1>, … ]`, URL-persisted via `?team=…`. Mirrors
  `src/nfl_filter.py` exactly.
- **Cron naming** — `scripts/daily_f1.py`. Schedule on Railway as
  its own service, daily, alongside `daily_nfl.py`.

## v1 ideas (deliberately not done now)

- Driver channels (Lando Norris, Max Verstappen, etc.) as a
  separate Z3 layer.
- Per-video ingestion (matches the WC2026 video pipeline).
- Per-race highlights / race-weekend timeline.
- Championship-table integration (live standings overlay).

Each is a natural extension of the v0 architecture — none requires
re-shaping what's in the DB now.
